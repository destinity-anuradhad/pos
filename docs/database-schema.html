TablePurpose

Tables present only in Cloud (PostgreSQL)
TablePurpose

Column-level differences on shared tables
Table · ColumnLocal (SQLite)Cloud (PostgreSQL)

tables · status_id FK → table_statuses.id — not present — (no status tracking)
categories/products/tables · sync_status pending / synced (on each row) — not present — (cloud is source of truth)
orders · hq_order_id Integer, nullable (set after sync) — not present — (cloud id IS the order id)
orders · sync_status / receipt_sent Present — not present —
orders · created_at created_at (when placed) order_created_at (original) + synced_at (when received)
orders · table reference table_id (FK) table_name (denormalized string snapshot)
orders · outlet_id — not present — FK → outlets.id
order_items · product_name nullable snapshot NOT NULL — required in cloud
order_items · product_id FK → products.id (nullable) Integer, nullable — no FK constraint
terminals · outlet_id / platform / registered_by platform, registered_by, registered_at, last_seen_at outlet_id (FK), last_sync_at
