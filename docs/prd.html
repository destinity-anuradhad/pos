1 Project Overview

Destinity Inspire POS is a guided, cross-platform Point of Sale application designed specifically for restaurants. It is built from a single shared codebase and deployable across Web, Windows, Linux, Android, and iOS. The app supports offline-first operation with automatic background sync to a central backend when connectivity is available.

Goals

  • Provide a simple, guided 4-page POS experience for restaurant staff.
  • Work seamlessly online and offline on all platforms.
  • Enable fast order entry with barcode and QR code scanning on mobile.
  • Support table management for dine-in orders.
  • Generate receipts via print, email/SMS, and in-app view.
  • Support dual-currency billing (LKR and USD).

Out of Scope (v1)

  • Multi-user / role-based access (planned for v2)
  • Kitchen display system (KDS)
  • Loyalty / rewards program
  • Advanced inventory management

4 Application Pages

The app consists of 4 core pages:

1. Dashboard
  • Daily sales total (LKR & USD)
  • Total orders today
  • Quick stats cards
  • Recent orders list
  • Online/offline status
2. Products
  • Product list with categories
  • Add / edit / delete product
  • Product name, price, barcode
  • Category management
  • Search products
3. POS / Cashier
  • Table selection
  • Product grid / search
  • Barcode & QR scan
  • Cart management
  • Currency toggle (LKR/USD)
  • Checkout & receipt
4. Orders
  • Sales history list
  • Filter by date
  • Order detail view
  • Re-print / resend receipt
  • Sync status indicator

5 Feature Requirements

5.1 Table Management

  • Restaurant floor view with numbered tables.
  • Each table shows status: Available, Occupied, or Billed.
  • Orders are assigned to a specific table.
  • Multiple orders per table session supported.

5.2 Barcode & QR Code Scanning

  • On Android/iOS: native camera via Capacitor Barcode Scanner plugin.
  • On Web/Desktop: webcam-based scanning as fallback.
  • Manual barcode entry always available as a fallback.
  • Supports both product barcodes and QR codes (e.g. QR payment).

5.3 Receipt Generation

MethodSupportedNotes
Print (thermal)YesBrowser/Electron print dialog
EmailYesSend via backend email service
SMSYesSend via SMS gateway
In-app viewYesDisplayed after checkout

5.4 Currency Support

  • Supports LKR (Sri Lankan Rupee) and USD (US Dollar).
  • Cashier can toggle currency at checkout.
  • Exchange rate can be set manually in settings.
  • Receipts display the selected currency.

5.5 Offline-First & Auto Sync

  • All transactions saved locally to SQLite first.
  • App fully functional without internet connection.
  • When connectivity is detected, pending data auto-syncs to backend.
  • Sync status indicator shown on Dashboard and Orders pages.
  • Conflict resolution: last-write-wins for product updates.

5.6 User Management (v1)

  • Single user — no login required in v1.
  • Multi-user with roles (Admin / Cashier) planned for v2.

6 Database Design (High Level)

SQLite is used both locally on the device and on the backend server.

Design Principles

  • Clean, minimal UI suitable for fast-paced restaurant environments.
  • Large touch targets for mobile and tablet use.

  • Consistent use of primary color for navigation and actions.
  • Secondary color for call-to-action buttons (checkout, confirm).
